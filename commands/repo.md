# Publication-Ready Repository Preparation

You are preparing a scientific repository to accompany a peer-reviewed manuscript. The repository will be scrutinized by reviewers, editors, and readers. It must look like a finished, professional product — not a workspace. Reviewers will use any sign of sloppiness, placeholder content, or AI tooling as ammunition against the manuscript.

## Setup

1. **Identify all repositories.** Ask the user which platforms host material for this manuscript (GitHub, HuggingFace, Zenodo, Figshare, Dryad, etc.). Get URLs or local paths for each.

2. **Identify the manuscript citation.** Ask for the full citation string (authors, title, journal if known). This will replace all vague references like "the accompanying manuscript."

3. **Clone or locate each repository locally.** For remote-only repos, clone them (use `GIT_LFS_SKIP_SMUDGE=1` for HuggingFace repos to avoid downloading large files).

## Audit Passes

Execute four passes across ALL repositories. Each pass produces a findings list before any edits are made.

### Pass 1: Work-in-Progress Language

Scan every text file (README, docs, comments, model cards, YAML frontmatter, docstrings) for language that signals the repository is unfinished:

- **Placeholder references:** "the accompanying manuscript," "manuscript in preparation," "paper in prep," "forthcoming"
- **Co-author coordination:** "please add," "co-authors should," "contributors please," "when adding files," "if you need to add"
- **Hedging qualifiers:** "(if needed)," "(TBD)," "(placeholder)," "(draft)," "(WIP)," "(temporary)"
- **Placeholder figure/table numbers:** "Figure X," "Table S?," "Supplementary ??"
- **TODO/FIXME/HACK comments** in any source file
- **Internal notes or instructions** not meant for public consumption

For each finding: note the file, line number, the offending text, and what it should become.

### Pass 2: AI Provenance Scrub

Reviewers and editors actively look for AI fingerprints in submitted repositories. Scan for:

- **Commit messages:** `Co-Authored-By` trailers referencing Claude, GPT, Copilot, or any AI tool. Also check for `noreply@anthropic.com`, `noreply@openai.com`, or similar.
- **Commit author/committer metadata:** Any author name or email that is not a human team member.
- **Contributors page:** GitHub derives contributors from commit metadata — even deleted trailers persist if old refs or dangling objects remain. After rewriting, ALWAYS purge `refs/original/`, expire reflogs, and `gc --prune=now`.
- **File contents:** Comments like "generated by Claude," "AI-assisted," etc. in source files.
- **Git blame:** Check that no commit in the history has a non-human author/committer identity.
- **Hidden files:** `.claude/`, `.copilot/`, `.cursor/`, `.aider/`, or any AI tool configuration that may have been accidentally committed.

The complete remediation sequence for commit history is:
```
git filter-branch -f --msg-filter 'sed "/Co-Authored-By:.*[pattern]/d"' -- --all
git update-ref -d refs/original/refs/heads/main
git update-ref -d refs/original/refs/remotes/origin/main
git reflog expire --expire=now --all
git gc --prune=now --aggressive
git push --force origin main
```
After force push, verify via `gh api repos/OWNER/REPO/contributors --jq '.[].login'` that only human accounts appear.

### Pass 3: Structural & Metadata Completeness

Check that each repository meets minimum standards for a publication companion:

- **README:** Present, substantive, not a template. Must include: title, one-paragraph description tied to the citation, repository structure, reproduction instructions, software/dependency list, citation block, license.
- **LICENSE file:** Present and appropriate (MIT, Apache-2.0, CC-BY-4.0 for data, etc.).
- **Citation block:** Every repository README must contain a Citation section with the manuscript reference. Use blockquote format.
- **Data provenance:** If the repo contains data or results, there should be documentation of how outputs were generated (chain of custody).
- **requirements.txt / environment.yml:** If code is present, dependencies must be pinned or at minimum version-bounded.
- **HuggingFace model cards:** Must have proper YAML frontmatter (language, license, tags, pipeline_tag). The description must reference the citation, not "the accompanying manuscript."
- **Zenodo metadata:** `.zenodo.json` or proper metadata if using Zenodo integration. Check that creators, title, description, and keywords are set.
- **Cross-references:** GitHub README should link to HuggingFace (and vice versa) if both exist. Same for Zenodo DOI badges.
- **No empty directories or placeholder files:** Remove any `__init__.py` that exists only to hold a directory, `.gitkeep` files in directories that now have content, or empty stub files.

### Pass 4: Code Hygiene

Scan source files for issues that signal an unfinished codebase:

- **Hardcoded paths:** `/home/username/`, `/scratch/`, `/Users/`, HPC-specific paths. All paths should be relative.
- **Dead imports:** Modules imported but never used.
- **Commented-out code blocks:** Large sections of commented code (a few lines for explanation is fine; 20+ lines of dead code is not).
- **Debug prints:** `print()` statements used for debugging that should be removed or replaced with proper logging.
- **Credentials or secrets:** API keys, tokens, passwords, `.env` files. These are both a security issue and a professionalism issue.

## Execution

After all four passes are complete, present the full findings list to the user organized by repository and pass. Then ask for confirmation before making changes.

When making changes:
- Edit files to fix all identified issues.
- For commit history rewrites, follow the full purge sequence in Pass 2.
- Commit with clean, professional messages authored by the user (never include AI co-author trailers).
- Push all changes.
- Verify the public-facing state of each repository after pushing (contributors list, README rendering, model card rendering).

## Final Verification Checklist

After all changes are pushed, run through this checklist and report status:

```
[ ] Every README references the manuscript by citation, not by vague pointer
[ ] No TODO/FIXME/HACK/WIP/TBD in any committed file
[ ] No AI tool references in commit history (messages, author, committer)
[ ] Contributors list shows only human team members
[ ] No hardcoded local/HPC paths in any script
[ ] License file present in every repository
[ ] Citation section present in every README/model card
[ ] Cross-links between companion repositories are correct and functional
[ ] Reproduction instructions are present and reference actual files that exist
[ ] No credentials, secrets, or .env files committed
[ ] No AI tool configuration directories (.claude/, .copilot/, etc.) committed
[ ] HuggingFace model card YAML frontmatter is complete
```

Report each item as PASS or FAIL with a note. Any FAIL items must be resolved before the repository is considered publication-ready.
