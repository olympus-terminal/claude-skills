Steps:
1. Parse the error message to identify:
   - Error type/category
   - File and line number if present
   - Stack trace components
2. Read the relevant source file(s) at the indicated locations
3. Analyze the code context to understand what went wrong
4. Explain the error in plain language:
   - What the error means
   - Why it occurred in this specific context
   - Common causes of this error type
5. Provide specific fix suggestions:
   - Minimal fix for the immediate issue
   - Better patterns to prevent similar errors
   - Related best practices
